# V3.1 experiment bundle (literature-aligned upgrades)

Parallel track to **V2**: new defaults, **forked Python entrypoints**, optional dataset QC, stronger domain-adaptation modes, and explicit **threshold policies** for final-test reporting. **V2 scripts and folders are unchanged.**

## Primary endpoints

- **Proxy objective:** Improve calibrated ranking / classification on Holstein `video_health_status` (or filtered manifest), with metrics that flag degenerate thresholding (all-positive / all-negative).
- **Scientific objective:** Ultimately align evaluation with veterinary pain or herd-health gold labels; until then, claims remain explicitly limited to the proxy (see repo `literature_review.md`).

## Literature mapping (see also `holstein_cv_v31_plan.md`)

| Idea | Reference |

## Quick start (repo root)

```bash
bash V3.1/run_task1_v3_1.sh
```

Optional: pooled validation F1-opt threshold (often reduces pathological fold-mean thresholds):

```bash
export EXTRA_DANN_ARGS="--test-threshold-policy pooled_val_f1_opt"
export EXTRA_WEAK_ARGS="--test-threshold-policy pooled_val_f1_opt"
bash V3.1/run_task1_v3_1.sh
```

Optional CDAN domain conditioning:

```bash
export EXTRA_DANN_ARGS="--da-mode cdan --domain-warmup-epochs 3 --domain-ramp-epochs 10"
bash V3.1/run_task1_v3_1.sh
```
