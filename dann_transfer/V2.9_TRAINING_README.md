# v2.9 Training (UCAPS) - v2.5-stable defaults + two-stage option

This document explains what `v2.9_training_classification.py` does and how to run it safely in Colab without overwriting previous runs.

## What v2.9 changes (vs v2.8 defaults)

v2.9 is meant to be "stability-first" for the small UCAPS regime:

- **Losses (default):**
  - Task1: `BCEWithLogitsLoss`
  - Task2: **plain CrossEntropy** with inverse-frequency weights (`--task2_loss_type ce`)
- **Shared moment weighting (v2.5 behavior):**
  - Default: `M0=1, M1=1, M2=4, M3=2, M4=1.2`
  - Applied to **both** tasks (no per-task split by default)
- **Temporal stability (default):**
  - `--temporal_sampling linspace`
  - `--time_reverse_p 0.0`
- **No EMA by default** (enable explicitly with `--ema`)
- **No label smoothing by default** (`--label_smoothing 0.0`)
- **Scheduler (v2.5-like):** `ReduceLROnPlateau(mode="min")` stepped on **val_loss**
- **Combined selection metric (default):**
  - Task1: fixed-threshold F1 (no threshold optimization by default)
  - Task2: **weighted F1** (matches v2.5 reporting)

## Safety: no-overwrite runs (run_tag)

Outputs go to:

- `checkpoints_v2.9/<run_tag>/`
- `results_v2.9/<run_tag>/`

If you do not pass `--run_tag`, v2.9 auto-generates a unique tag (timestamped) to avoid overwriting.

If a run folder already contains artifacts, the script refuses to write unless:

- `--overwrite_run` is passed, or
- the folder contains a resume checkpoint and `--auto_resume` is enabled.

## Safety: leakage checks (folds)

Before training, v2.9 runs a hard-fail leakage check by default:

- test animals do not appear in any fold train/val
- train/val animal overlap within a fold is empty
- clip overlap (sequence_path / clip_id) between train and val is empty
- test clips do not appear in any train/val split

To only run the checks and exit:

```bash
!python Ucaps_raw_videos/v2.9_training_classification.py --check_splits
```

To skip (not recommended):

```bash
!python Ucaps_raw_videos/v2.9_training_classification.py --skip_split_check
```

## Holstein weak-label cow-held-out adaptation

`weak_label_adapt_v2.9.py` fine-tunes/evaluates **Task1 only** on the Holstein/Jersey transfer set by treating `video_health_status` as a weak binary proxy:

- `Healthy` -> no-pain proxy (`0`, encoded as UCAPS `M0`)
- `Unhealthy` -> pain proxy (`1`, encoded as UCAPS `M2`)

The protocol keeps **4 cows as a final test set**, then uses the remaining cows for inner CV with **4 validation cows per fold**. This avoids cow identity leakage across train/validation/test.

First verify the split:

```bash
python weak_label_adapt_v2.9.py \
  --manifest-csv ../cow_face_sequences_10s_250/completed_manifest.csv \
  --sequence-root ../cow_face_sequences_10s_250 \
  --out-dir holstein_weak_label_cv_outputs \
  --label-column video_health_status \
  --test-cows 4 \
  --val-cows-per-fold 4 \
  --dry-run
```

Then run a conservative fine-tune from UCAPS weights:

```bash
python weak_label_adapt_v2.9.py \
  --manifest-csv ../cow_face_sequences_10s_250/completed_manifest.csv \
  --sequence-root ../cow_face_sequences_10s_250 \
  --checkpoint-dir checkpoints_v2.9/v2.9_20260222_144752 \
  --ckpt-kind task2 \
  --init-fold 0 \
  --out-dir holstein_weak_label_cv_outputs \
  --label-column video_health_status \
  --test-cows 4 \
  --val-cows-per-fold 4 \
  --num-epochs 10 \
  --batch-size 8 \
  --freeze-cnn
```

Main outputs:

- `weak_label_cv_splits.json`
- `weak_label_cv_fold_summary.csv`
- `weak_label_cv_predictions.csv`
- `weak_label_cv_test_predictions.csv`
- `weak_label_cv_summary.json`
- `weak_label_cv_report.md`

These are still **weak disease-context proxy** results, not veterinary-validated pain scores.

## Holstein self-supervised + DANN adaptation

The next transfer-learning layer is intentionally additive: keep the zero-shot and weak-label fine-tune baselines above, then compare self-supervised pretraining and domain-adversarial adaptation against the exact same cow-held-out split.

`ssl_pretrain_holstein_v2.9.py` uses SimSiam over two sequence-consistent augmented views of each Holstein/Jersey clip. By default, each fold pretrains only on that fold's **training cows**, so validation/test cows do not leak into representation learning.

```bash
python ssl_pretrain_holstein_v2.9.py \
  --manifest-csv ../cow_face_sequences_10s_250/completed_manifest.csv \
  --sequence-root ../cow_face_sequences_10s_250 \
  --checkpoint-dir checkpoints_v2.9/v2.9_20260222_144752 \
  --ckpt-kind task2 \
  --init-fold 0 \
  --out-dir holstein_ssl_outputs \
  --label-column video_health_status \
  --test-cows 4 \
  --val-cows-per-fold 4 \
  --num-epochs 20 \
  --batch-size 8
```

Useful quick check:


### 2) DANN source-target adaptation

`dann_adapt_v2.9.py` trains labeled UCAPS source batches and Holstein/Jersey target batches together. It adds:

- `GradientReversalLayer`
- `DomainClassifier`
- source Task1 BCE
- optional source Task2 CE
- domain CE over source vs target embeddings
- optional late, low-weight target weak BCE

Dry-run the target split first:

```bash
python dann_adapt_v2.9.py \
  --manifest-csv ../cow_face_sequences_10s_250/completed_manifest.csv \
  --sequence-root ../cow_face_sequences_10s_250 \
  --out-dir holstein_dann_outputs_dry_run \
  --dry-run
```

Then train DANN. Replace the UCAPS paths with the project folder that contains `train_val_test_splits_v2.json`, `sequence_label_mapping_v2.json`, and the matching `sequence/` folder:

```bash
python dann_adapt_v2.9.py \
  --manifest-csv ../cow_face_sequences_10s_250/completed_manifest.csv \
  --sequence-root ../cow_face_sequences_10s_250 \
  --source-project-dir /path/to/facial_pain_project_v2 \
  --source-sequence-dir /path/to/sequence \
  --checkpoint-dir checkpoints_v2.9/v2.9_20260222_144752 \
  --ckpt-kind task2 \
  --init-fold 0 \
  --ssl-checkpoint-dir holstein_ssl_outputs \
  --out-dir holstein_dann_outputs \
  --label-column video_health_status \
  --test-cows 4 \
  --val-cows-per-fold 4 \
  --num-epochs 20 \
  --batch-size 8 \
  --learning-rate 1e-5 \
  --domain-weight 0.5 \
  --source-task2-weight 0.2 \
  --target-weak-weight 0.0
```

Main outputs:

- `dann_splits.json`
- `dann_run.json`
- `dann_fold_summary.csv`
- `dann_predictions.csv`
- `dann_test_predictions.csv`
- `dann_test_cow_aggregates.csv`
- `dann_summary.json`
- `dann_report.md`
- `fold_<n>/best_dann.pt`
- `fold_<n>/history.csv`

Interpretation guardrails:

- Keep the main DANN run at `--target-weak-weight 0.0` first; add weak target BCE only as a later ablation, for example `--target-weak-weight 0.1 --target-weak-start-epoch 5 --ramp-target-weak`.
- Watch `domain_acc` in each fold history. Near 1.0 means the embedding still separates source vs target; source metrics collapsing means the adversarial loss is too strong.
- These outputs still evaluate weak disease-context labels, not veterinary pain scores.

## How to improve from here (highest-yield next experiments)

Note: the error analysis and recommended next experiments below were written for the strongest held-out test run so far (`v2.9_20260221_014705`, joint training). The two-stage run (`v2.9_20260222_144752`) currently shows a Task2 collapse on the held-out animals and should be debugged via per-fold test outputs before drawing conclusions from its test metrics.

Your Task2 failure mode is clear: **Residual (M3/M4) is being predicted as Acute (M2)** (10/12 residual clips -> class 1).

From `test_eval_v2.9_combined_animals_14-17_predictions.csv` for this run:

- Residual (true=2, N=12): predicted 2 for **2/12**, predicted 1 for **10/12**
  - Animal 17: M3 residual predicted correctly (2/2), but M4 residual predicted as Acute (3/3)
  - Animal 14: all residual (M3/M4) predicted as Acute (7/7)
- No-pain (true=0, N=14): predicted 0 for **11/14**, predicted 1 for **3/14**
- Acute (true=1, N=9): predicted 1 for **9/9** (high recall, low precision due to residual/no-pain -> acute leakage)
